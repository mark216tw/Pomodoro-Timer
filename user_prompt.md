# 使用者自訂提示詞 (User Custom Prompt)

專案名稱：好用的番茄時鐘

你現在是一位資深前端工程師 + UI/UX 設計師，請幫我製作一個純前端、單一 HTML 檔案的番茄時鐘網站，系統名稱為：「好用的番茄時鐘」

# 一、整體目標
- 製作一個 可直接開啟即可使用（無需後端） 的番茄時鐘
- 所有HTML代碼寫在單一 index.html檔案、所有CSS代碼寫在單一 style.css檔案 、 所有javascript代碼寫在單一 script.js檔案  
- 使用者體驗要「簡單、專注、好用」
- 程式碼需結構清楚、可讀性高，適合後續自行修改
- 將操作流程寫入  README.md 

# 二、功能需求（Functional Requirements）
## 1️⃣ 基本番茄時鐘功能
- 預設工作時間：25 分鐘
- 預設休息時間：5 分鐘
- 顯示目前狀態：工作中（🍅 專注）、休息中（☕ 休息）

## 2️⃣ 操作按鈕
- ▶️ 開始
- ⏸ 暫停
- 🔁 重置

## 3️⃣ 時間顯示
- 顯示格式：MM:SS
- 倒數過程即時更新
- 當時間結束時：
  - 自動切換「工作 / 休息」狀態
  - 發出提示音（可使用 HTML5 Audio）

## 4️⃣ 狀態提示
- 清楚顯示：
  - 目前是「第幾個番茄鐘」
  - 目前模式（工作 / 休息）
- 番茄完成時顯示視覺提示（如顏色變化）

## 5️⃣ 瀏覽器支援
 - 支援 Chrome / Edge / Firefox
 - 不使用任何外部套件或框架

# 三、技術限制與說明（Technical Requirements）
## 📄 檔案結構
- 包含 index.html 、style.css 、 script.js 

## 🧱 技術規範
- HTML5
- CSS3（寫在 style.css 檔案）
- JavaScript（寫在 script.js 檔案）

## ❌ 不可使用：
- React / Vue / jQuery
- CDN / 外部 JS / CSS
- Build tool

## 🧠 JavaScript 實作要求
- 使用 setInterval / clearInterval
- 明確區分：
  - 狀態管理
  - 計時邏輯
  - UI 更新
- 重要變數與函式需有清楚命名
- 關鍵邏輯請加上註解說明

# 四、介面設計說明（UI / UX）
## 🎨 整體風格
- 簡約、專注、不干擾
- 使用柔和背景色
- 中央顯示倒數時間（視覺焦點）

## 🧩 介面配置
- 頁面置中
- 主要元素：
1.- 系統標題：「好用的番茄時鐘」
2. 當前模式文字（工作 / 休息）
3. 大型倒數時間顯示
4. 控制按鈕區（開始 / 暫停 / 重置）
5. 番茄完成數顯示
6. 配色請參考莫蘭迪色系
7. 支援淺色模式與深色模式
8. 提供可以指定工作時間 (分鐘)、休息時間 (分鐘)功能 

## 📱 響應式設計
- 桌機與手機皆可正常使用
- 使用 Flexbox 或 Grid

# 五、加分項目（Optional but Recommended）
- 使用 localStorage 儲存已完成的番茄數量，並提供「刪除儲存」功能，可以刪除 localStorage 儲存，刪除前與使用者確認，確認功能不要使用瀏覽器內建對話框（window.confirm），使用自定義的彈窗 (Modal)。
- localStorage 儲存使用者選擇的淺色模式與深色模式 
- 番茄完成時播放音效（可內嵌 base64 音效），音效要好聽一點， 並且提供播放音效的開關
- 不同模式切換時改變背景色
- CSS 動畫（如倒數閃爍）
- 提供5個主題供使用者選擇

# 六、輸出格式要求（Important）
- 不要省略任何必要內容
- 不要附加解說文字在 HTML 外

# 七、程式品質要求
- 程式碼需：
  - 可讀
  - 易維護
  - 命名清楚
- 請確保：
  - 不會產生多重計時器
  - 暫停 / 重置邏輯正確
  - 切換模式不會錯亂

🚀 請直接開始產生完整程式碼
